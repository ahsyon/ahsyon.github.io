我应该使用OpenCore还是Clover

截止到目前为止，无论是OpenCore还是Clover，都已经支持Ventura安装使用了

  • OpenCore v0.9.5 下载
    • OCAT 下载
  • Clover v5148 下载
    • Clover Configurator v5.18.3.0 下载

我可以直接在线升级吗

我给您的建议是：不建议小白直接在线更新；比较稳妥的方法是在现在的磁盘上切一个单独的分区进行安装测试

如何保证顺利升级 macOS 正式版

  • 请使用OpenCore v0.9.5 下载 / Clover v5148 下载
  • 请于boot-args中添加引导参数：vsmcgen=1，
    • 可能还需要其它引导参数：-lilubetaall vsmcbeta
    • DW1820A / DW1560 / DW1830等博通网卡可能还需要添加：brcmfx-aspm

AMD 6000 系列显卡当前 macOS 状态：

  • 在 macOS 上工作的显卡
    • Family Navi21：6800 / 6800XT / 6900XT / 6950XT（自 Big Sur 11.4 起）
    • Family Navi23：6600 / 6600XT / 6600M / 6600S / 6650M / 6650XT / 6700S / 6800S（自 Monterey 12.1 起）
  • 不适用于 macOS
    • Family Navi22：6700M / 6700XT / 6750XT / 6800M / 6850M XT
    • Family Navi24：6400 / 6500 和 6500XT

安装程序进入 Recovery / Install macOS Ventura 灰屏，只出现一个鼠标的解决方法

修改 config.plist，Nvram -> 7C436110-AB2A-4BBB-A880-FE41995C9F82 -> prev-lang:kbd -> en-US:0，原来的参数有可能为空了导致的；重启后执行 reset NVRAM 的动作后就可以正常进入安装界面了

Intel I225-V 驱动方法

Monterey提取驱动：位置 /System/Library/Extensions/IONetworkingFamily.kext/Contents/PlugIns，复制 AppleIntelI210Ethernet.kext 到 /EFI/OC/Kexts，然后配置文件里加载它，收工。

  • 下载：BrcmPatchRAM-2.6.4-RELEASE.zip

  • 在配置文件 config.plist 中添加BlueToolFixup.kext 驱动，做好内核版本控制，如下图所示：

BIOS未禁用CFG LOCK选项的设置

相关错误信息：

  • 卡在 [EB|#LOG:EXITBS:START]

某些用户会忘记或无法禁用BIOS中的CFG-Lock（特别是与用于电源管理的锁定0xE2 MSR位有关，显然更安全地关闭CFG-Lock）。**请注意，这仅适用于Intel用户，不适用于AMD。**发生这种情况时，有几个可能的解决方法：

打开OpenCore的配置文件config.plist，勾选如下三个选项：

  • Kernel -> Quirks

    • [x] AppleCpuPmCfgLock
    • [x] AppleXcpmCfgLock

  • UEFI -> Quirks

    • [x] IgnoreInvalidFlexRatio

开机卡在apfs_module_start: 1689: load: com.apple.filesystems.apfs, v1412.101.1, apfs-1412. 101.1 附近

解决方法：

打开配置文件：config.plist -> UEFI-UEFI设置

  • [x] ReleaseUsbOwnership

BIOS重设或重启/关机后发送到安全模式

AppleRTC的问题，很简单的解决方法：

  • 打开配置文件：config.plist
    • [x] Kernel -> Quirks -> DisableRtcChecksum

注意：如果仍然有问题，则需要使用RTCMemoryFixup.kext并排除范围。
